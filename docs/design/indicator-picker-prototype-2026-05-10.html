Indicator picker · prototype

docs/design/indicator-picker-prototype-2026-05-10.html

Multi-instance picker for the right rail (~360 px). Three frames below show the three states called out in the brief — empty, filtered, hover-preview. The component itself is everything inside .ip-root; every rule is namespaced so it ports to an Angular standalone component as-is.

State A · Empty /data-lab

No facets active · all 4 categories collapsed

State B · Filtered /research-lab/feature-runner

Pane = Sub-panel · Category = Momentum · 1 instance staged

State C · Hover preview /research-lab/signal-runner

Hovered rsi · popover shown after 300 ms

Inputs · Outputs

  • [indicators] IndicatorInfo[] from IndicatorCatalogService
  • [activeKeys] string[] — drives the +N badge per row
  • [presets] IndicatorPreset[] — host-supplied
  • (add) · (addInstance) · (preview)

Facets (orthogonal)

  • Pane: Overlay / Sub-panel — independent toggles, both-off = both shown.
  • Category: trend · momentum · volatility · volume — multi-select, both-off = all.
  • Counts on category cards reflect the filtered set, not the catalog.

Preview behaviour

  • Pointer rests on row → 300 ms timer → popover anchored to row, opens to the left so it never clips the right edge of the rail.
  • Pure mouse-out closes immediately; clicking Add dismisses it as a side effect.
  • Mini-chart is a static sine-with-noise sample in this prototype — the Angular component will swap in service-fed data on the same canvas.
