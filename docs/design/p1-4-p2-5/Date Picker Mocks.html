LEAN-Lab date picker — blocked-aware states

Mocks for the LEAN-Lab date picker after the P2.5 semantic shift to half-open [09:30 ET start, 09:30 ET next_trading_day(end)) windows. Picker chrome mirrors the engine-lab ticker-range-picker — eyebrow group cards, mono summary line, day-availability strip, smart-legend chips, and an advisories cluster. New surface: a popover month grid that disables weekends, holidays, and half-days against the calendar endpoint, with DST-boundary indicators and contextual rejection tooltips. Six states, dark theme, exact token palette.

Interaction states

Each frame shows the picker in a different state. The collapsed input row is the resting state; the popover opens on focus and stays open while the user drags a range.

Design decisions encoded

What this mock locks in beyond what the v2 handoff specifies. Implementation agent should treat these as the picker contract.

D1
Native input → custom popover
<input type="date"> can't disable specific dates per the v2 constraint. Picker swaps to a custom month grid backed by the /blocked-dates endpoint. Native input remains the resting visual but opens the popover on focus.
D2
Three disabled-state glyphs
Weekends are recessed bg only (no glyph). Holidays get a small H corner badge. Half-days get a ½ badge plus warn-tinted diagonal stripe. Half-days are the visually loudest because they're the new constraint.
D3
Tooltip = teaching surface
On hover of any disabled cell, the tooltip names the cause AND the contract. Copy mirrors the server's HTTP 400 message verbatim — when a request slips through client validation, users see the same words twice and learn the rule.
D4
DST is info, not warn
DST boundaries are valid windows — they just shift the UTC-ms boundary. Info-color advisory + boundary marker on the strip. Calling it warn would imply the user is doing something wrong.
D5
Strip cell = trading session
v1 engine-lab strip used one cell per calendar day (weekends included, dimmed). Under the P2.5 contract the window is sessions-only, so the strip renders one cell per session — keeps the dominant-legend math honest.
D6
Show-computed-window detail
State F's expandable detail is the auditability seam. Users debugging a manifest mismatch can open it to see exactly which ms went on the wire. Cheap to add and worth a lot when a cross-engine reconcile disagrees on the trading-date set.
