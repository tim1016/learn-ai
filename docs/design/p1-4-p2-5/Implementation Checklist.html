Implementation checklist — workspace cap & date semantics

File-by-file plan extracted from 2026-05-18-design-p1-4-live-workspace-cap-v2.md and 2026-05-18-design-p2-5-date-semantics-v2.md. Steps are ordered for least-rebase; tests are listed last in each PR but should land in the same commit that introduces the surface. Anything called out under Watch out for is a latent silent-correctness bug the v2 docs flagged.

Source of truth: v2 design handoffs on master PRs: 2 (independent) Reviewers: CodeRabbit + Codex
PR · P1.4

Live workspace cap — background poller

single PR no upstream deps
Threat model — load-bearing. This cap defends against benign overrun (a buggy QCAlgorithm filling its own disk), not adversarial input. The data plane is a single-host research tool with no caller auth; if auth is added later and algorithm_source_kind=user_provided starts arriving from a genuinely untrusted caller, escalate to kernel-enforced (--storage-opt size=<mb>m). The post-execute backstop is not sufficient against an adversary. Encode this paragraph in the poller class's module docstring.
1

Add the poll-interval constant

PythonDataService/app/lean_sidecar/config.py
_WORKSPACE_POLL_INTERVAL_S = 1.0 as a module-level constant. Not exposed in RunLimits.
Watch: do not add a RunLimits field for this. The interval is the overshoot budget; making it caller-settable widens it unsafely.
2

Add the kill-reason enum

PythonDataService/app/lean_sidecar/runner.py
class KillReason(StrEnum) with WALL_CLOCK_TIMEOUT and WORKSPACE_MAX_MB_EXCEEDED. Co-locate with the cidfile helper — they're consumed together.
3

Thread reason through the kill helper

PythonDataService/app/lean_sidecar/runner.py
_kill_container_via_cidfile(reason: KillReason). Log it, include it in the kill-event payload, and route it directly into RunResult.rejection_reason.
Watch: no string magic. PR #280's existing call site passes KillReason.WALL_CLOCK_TIMEOUT explicitly — don't default it.
4

Wire the poller thread around execute()

PythonDataService/app/lean_sidecar/launcher/service.py
Background thread runs while execute() is in-flight. Walks the workspace with os.scandir (recursive, C-backed stat) — not Path.rglob. On overrun: call _kill_container_via_cidfile(reason=KillReason.WORKSPACE_MAX_MB_EXCEEDED). A threading.Event stops the poller on normal exit.
Watch: if scandir-loop is still expensive on a 100k-file workspace, fall back to subprocess.run(["du", "-sb", workspace_path]). Target: poller iteration ≤50ms.
5

Encode the threat model in the docstring

PythonDataService/app/lean_sidecar/launcher/service.py
Module-level (or poller-class) docstring carries the benign-overrun-not-adversarial paragraph verbatim. Reviewers and the next agent re-derive nothing.

Tests — PythonDataService/tests/lean_sidecar/test_runner.py

  • Happy path — small write, poller never trips, normal exit.
  • Overrun path — write exceeds cap mid-execute; kill helper called with KillReason.WORKSPACE_MAX_MB_EXCEEDED; RunResult.rejection_reason matches.
  • Race path — write lands JUST as execute() exits; post-execute backstop still catches it.
  • Walk-cost smoke — 10k–100k synthetic files; one poller iteration ≤50ms (with generous margin).
  • No regression on PR #280 timeout-kill path — KillReason.WALL_CLOCK_TIMEOUT still flows end-to-end.

Out of scope for this PR

  • Kernel-enforced cap (--storage-opt size=) — deferred until auth lands.
  • tmpfs mount — bind mount is load-bearing for trusted-sample artifacts.
  • New RunLimits field for poll interval — constant only.
PR · P2.5

Date-window semantics — half-open session ms

single PR no upstream deps touches FE + BE
Contract. start_ms_utc = 09:30 ET of start_date. end_ms_utc = 09:30 ET of next_trading_day(end_date). Half-open [start, end). Validator rejects HTTP 400 if either endpoint isn't a trading day, or if any date in the inclusive range is an early-close (half-day). DST resolves through pandas_market_calendars' tz-aware machinery — never a fixed offset.
1

New module — single calendar source of truth

PythonDataService/app/lean_sidecar/trading_calendar.py
Surface:
  • is_trading_day(d: date) -> bool
  • is_early_close(d: date) -> bool
  • next_trading_day(d: date) -> date
  • session_open_ms_utc(d: date) -> int
Backed by _CALENDAR = mcal.get_calendar("NYSE") at module load (mirrors nyse_calendar.py's pattern). Both validator and staging iteration import from here — that's the load-bearing change.
Watch: do not merge with app/engine/live/nyse_calendar.py. Different consumers, different test surfaces, different rate of change.
2

Validator — reject weekends / holidays / half-days

PythonDataService/app/routers/lean_sidecar.py
On TrustedRunRequestModel.start_ms_utc/end_ms_utc:
  • Derive start_date, end_date from input ms.
  • Compute expected session_open_ms_utc(start_date) and session_open_ms_utc(next_trading_day(end_date)); reject on mismatch.
  • Reject if not is_trading_day(d) or is_early_close(d) for any d in [start_date, end_date].
Error messages are operator-readable and name the offending date.
Watch: the validator-before-staging invariant. If a refactor inverts it (staging-driven validation), half-day rejection becomes silent corruption.
3

Staging — iterate via the calendar module

PythonDataService/app/services/lean_sidecar_service.py (~152)
Convert session-boundary ms back to trading dates; iterate via trading_calendar.is_trading_day. Drop the existing +1 day hack — half-open ms already encodes the boundary.
4

Manifest — bump schema, annotate semantics

PythonDataService/app/lean_sidecar/manifest.py
Bump MANIFEST_SCHEMA_VERSION (per D10). Add to notes: date_semantics=session_open_half_open. Old manifests stay readable on the prior schema_version; the reconciler reads the note to know which contract applies.
5

Phase 5b template — recompute the hardcoded range

PythonDataService/app/lean_sidecar/templates/…
The 2025-01-06 → 2025-01-10 template needs start_ms_utc / end_ms_utc recomputed under the new contract. Full trading days, no half-days, no DST — value-only update, no semantic risk.
6

Backend — calendar endpoint for the UI

PythonDataService/app/routers/lean_sidecar.py
GET /api/lean-sidecar/calendar/blocked-dates?from=&to=. Returns the union of weekends, holidays, and half-days in the range. Single source of truth — no client-side duplication of the calendar.
Watch: this is the calendar-drift seam. Cache it sensibly on the client — don't let it become a per-pixel API call as the user navigates months in the picker.
7

UI — rename converter, fix DST conversion

Frontend/src/app/components/lean-lab/lean-lab.component.ts (~669)
isoDateToMsUtc → isoDateToSessionOpenMsUtc. Uses Intl.DateTimeFormat with timeZone: 'America/New_York'. No new dependency.
Watch: formatToParts returns strings — assemble the UTC Date carefully or DST days silently mis-convert by an hour. UI test surface MUST include the same DST dates as the Python test surface.
8

UI — blocked-aware date picker

Frontend/src/app/components/lean-lab/lean-lab.component.html + .ts
Replace the native <input type="date"> rows with a custom popover that disables weekends, holidays, and half-days. See Date Picker Mocks.html — 6 states with the engine-lab visual vocabulary (eyebrow groups, mono summary, availability strip, smart-legend chips, advisories cluster).
Watch: picker chrome should match shared/ticker-range-picker's time-window-card verbatim where possible (group card frame, eyebrow + dot, range row, presets, strip, legend). Reuse the SCSS partial.
9

UI — types comment update

Frontend/src/app/services/lean-sidecar.types.ts
start_ms_utc / end_ms_utc doc comment updates to reference the new contract. Field stays number; no wire-type change.
10

Cross-reconciler — assertion test only

PythonDataService/app/lean_sidecar/cross_reconciler.py
No logic change — it already derives NY trading date from each bar's ms. Add a test asserting requested_window_ms is now directly comparable to the derived per-bar trading dates.

Tests — mandatory surface (validator + UI)

  • Full-day window, all trading days → accept.
  • Window touching a weekend → reject; message names the weekend day.
  • Window touching a US federal holiday → reject; message names the holiday.
  • Window touching a half-day (e.g., 2026-11-27 Black Friday) → reject; message names the half-day.
  • DST start (2026-03-08 inside range) → session-open ms reflects EDT.
  • DST end (2026-11-01 inside range) → session-open ms reflects EST.
  • Round-trip — client sends, server validates, manifest persists, reconciler reads — assert all four agree on the trading-date set.
  • UI mirror — isoDateToSessionOpenMsUtc agrees with Python session_open_ms_utc on every DST date.

Out of scope for this PR

  • Wire-type change — start_ms_utc / end_ms_utc stay int. Semantic shift only.
  • Old-manifest migration — old runs stay readable on the prior schema_version.
  • Half-day support — validator rejects, UI disables. If needed later, relax the rejection; is_early_close is already in the calendar module.
  • Merging nyse_calendar.py into trading_calendar.py.
