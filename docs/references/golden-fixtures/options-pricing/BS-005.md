# BS-005 — Black-Scholes Call Theta

**Status:** active · v1 (2026-05-08)
**Category:** options-pricing
**Canonical:** `PythonDataService/app/services/bs_greeks.py::black_scholes_greeks`

## What this pins

Call theta from `black_scholes_greeks(...).theta`, validated against py_vollib==1.0.1 across 180 cases. Theta is the rate of change of option value with respect to calendar time.

## Oracle

`py_vollib.black_scholes.greeks.analytical.theta(flag='c', S, K, t, r, sigma)` — GPL-licensed external library, generation-only.

## Formula

```
Theta (call) = [-S·σ·e^(-qT)·N'(d1)/(2√T) - r·K·e^(-rT)·N(d2) + q·S·e^(-qT)·N(d1)] / 365

d1 = [ln(S/K) + (r - q + σ²/2)·T] / (σ·√T)
d2 = d1 - σ·√T
```

Division by 365 converts to per-calendar-day units.

## Units

Per calendar day. A theta of -0.05 means the option loses approximately $0.05 per calendar day. py_vollib and canonical use identical units — ratio=1.000000 verified across all 180 cases.

Values are **negative** for long calls (time decay is a cost to the holder).

## Agreement

Max observed abs error: < 1e-15.
Pinned tolerance: `atol=1e-10, rtol=0.0`.

## Additional Checks in Test Suite

- Theta is negative for all 180 call cases.
- ATM |theta| > deep-OTM |theta| (time decay is largest at-the-money).

## Known Limitations

- Call only. Put theta differs; BS-005b is a future fixture.
- Zero-dividend (q=0).
- No near-zero-TTM cases (theta magnitude blows up near expiry).
