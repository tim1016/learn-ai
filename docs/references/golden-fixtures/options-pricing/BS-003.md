# BS-003 — Black-Scholes Call Delta

**Status:** active · v1 (2026-05-08)
**Category:** options-pricing
**Canonical:** `PythonDataService/app/services/bs_greeks.py::black_scholes_greeks`

## What this pins

The call delta `Δ = e^(-qT)·N(d1)` from `black_scholes_greeks(...).delta`, validated against py_vollib==1.0.1 analytical delta as an independent oracle across the same 180-case input grid as BS-001/002.

## Oracle

`py_vollib.black_scholes.greeks.analytical.delta(flag='c', S, K, t, r, sigma)` — GPL-licensed external library, generation-only.

## Unit Verification (2026-05-08)

py_vollib delta and our canonical delta are identical values — both return the raw `N(d1)·disc_q` for calls, `(N(d1)-1)·disc_q` for puts. No unit conversion needed.

## Agreement

Max observed abs delta error: < 1e-15.  
Pinned tolerance: `atol=1e-10, rtol=0.0`.

## Argument Order Note

`black_scholes_greeks` argument order: `(spot, strike, ttm_years, volatility, rate, dividend, is_call)` — note `volatility` before `rate`, opposite of `bs_european_price`.

## Input Grid

180-case grid (same as BS-001), `is_call=True`.

## Additional Checks in Test Suite

- Call delta must be in (0, 1) for all cases.
- Deep ITM delta > ATM delta (monotonicity spot-check).

## QuantLib Canary

QuantLib delta canary is pending separate cross-engine verification. The existing `test_bs_cross_engine_parity.py` covers price parity; delta parity is pinned here against py_vollib only.
