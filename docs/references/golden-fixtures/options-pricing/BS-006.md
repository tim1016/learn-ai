# BS-006 — Black-Scholes Call Vega

**Status:** active · v1 (2026-05-08)
**Category:** options-pricing
**Canonical:** `PythonDataService/app/services/bs_greeks.py::black_scholes_greeks`

## What this pins

Call vega from `black_scholes_greeks(...).vega`, validated against py_vollib==1.0.1 across 180 cases. Vega is the sensitivity of option value to a 1-percentage-point change in implied volatility.

## Oracle

`py_vollib.black_scholes.greeks.analytical.vega(flag='c', S, K, t, r, sigma)` — GPL-licensed external library, generation-only.

## Formula

```text
Vega = S · e^(-qT) · N'(d1) · √T / 100

d1 = [ln(S/K) + (r - q + σ²/2)·T] / (σ·√T)
N'(x) = (1/√(2π)) · e^(-x²/2)
```

Division by 100 converts from per-unit-vol to per-1%-vol-move units.

## Units

Per 1%-volatility move. A vega of 0.20 means the option gains approximately $0.20 per 1 percentage point rise in implied volatility. py_vollib and canonical use identical units — ratio=1.000000 verified across all 180 cases.

Values are **positive** for long options (higher vol increases option value).

## Agreement

Max observed abs error: < 1e-15.
Pinned tolerance: `atol=1e-10, rtol=0.0`. Rationale: the canonical and py_vollib both evaluate the closed-form N'(d1) · √T / 100 expression using double-precision transcendentals; the observed max error of < 1e-15 confirms they use identical paths, and 1e-10 provides five orders of magnitude of headroom while remaining tight enough to catch any formula drift.

## Additional Checks in Test Suite

- Vega is positive for all 180 cases.
- Longer-dated ATM options have higher vega than shorter-dated ATM options.

## Known Limitations

- Call only (vega is identical for puts by put-call parity).
- Zero-dividend (q=0).
- No near-zero-TTM cases (vega → 0 near expiry).
