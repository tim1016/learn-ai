# BS-007 — Black-Scholes Call Rho

**Status:** active · v1 (2026-05-08)
**Category:** options-pricing
**Canonical:** `PythonDataService/app/services/bs_greeks.py::black_scholes_greeks`

## What this pins

Call rho from `black_scholes_greeks(...).rho`, validated against py_vollib==1.0.1 across 180 cases. Rho is the sensitivity of option value to a 1-percentage-point change in the risk-free rate.

## Oracle

`py_vollib.black_scholes.greeks.analytical.rho(flag='c', S, K, t, r, sigma)` — GPL-licensed external library, generation-only.

## Formula

```
Rho (call) = K · T · e^(-rT) · N(d2) / 100

d1 = [ln(S/K) + (r - q + σ²/2)·T] / (σ·√T)
d2 = d1 - σ·√T
```

Division by 100 converts from per-unit-rate to per-1%-rate-move units.

## Units

Per 1%-rate move. A rho of 0.10 means the option gains approximately $0.10 per 1% rise in the risk-free rate. py_vollib and canonical use identical units — ratio=1.000000 verified across all 180 cases.

Values are **positive** for long calls (higher rates increase call value via forward price).

## Agreement

Max observed abs error: < 1e-15.
Pinned tolerance: `atol=1e-10, rtol=0.0`.

## Additional Checks in Test Suite

- Call rho is positive for all 180 cases.
- Deep-ITM call rho > OTM call rho (more equity-like).

## Known Limitations

- Call only. Put rho is negative; BS-007b is a future fixture.
- Zero-dividend (q=0).
- Single risk-free rate [0.05]; rate-level sensitivity not swept.
