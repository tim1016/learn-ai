# IND-002 — Simple Moving Average (period=3)

**Status:** active · v1 (2026-05-09)
**Category:** indicators
**Canonical:** `PythonDataService/app/engine/indicators/sma.py::SimpleMovingAverage`

## What this pins

SMA values from `SimpleMovingAverage` (period=3), validated against an independent pure-Python rolling-window formula across 3 synthetic 8-bar price series (same cases as IND-001).

## Oracle

hand_computed — list-based rolling window without calling `SimpleMovingAverage`. Growing window during warmup, fixed window post-warmup.

## Formula

```
Warmup (samples 1..period-1): SMA = mean(prices[0..i])  — growing window
Post-warmup (samples period+): SMA = mean(prices[i-period+1..i])  — fixed window
```

Rolling window maintained as `deque(maxlen=period)` matching LEAN's `RollingWindow<T>(period)`.

## Reference

LEAN `Indicators/SimpleMovingAverage.cs` (vendored commit `7986ed0aade3ae5de06121682409f05984e32ff7`). Rolling window of maxlen=period; during warmup returns mean of all samples seen.

## Hand-Verification (Case A: prices=[10.0, 12.0, 14.0, 16.0, …])

| sample | SMA |
|--------|-----|
| 1 | 10.0 |
| 2 | 11.0 |
| 3 | 12.0 (is_ready, full window) |
| 4 | 14.0 |

## NaN Convention

All bars produce a float value. The SMA returns a value during warmup (growing window), so there are no `None` outputs.
