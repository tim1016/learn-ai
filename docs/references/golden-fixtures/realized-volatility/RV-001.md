## What this pins

Rolling close-to-close realized volatility on a 30-bar synthetic price series with window=10.

## Oracle

Pure-Python loop computing rolling `var(log_ret, ddof=1) × 252`, sqrt. Requires exactly `window` non-NaN log returns in each window — matching `pd.rolling(window, min_periods=window).var()` behavior. Since `log_ret[0] = NaN`, the first valid bar is index 10 (not 9).

## Agreement

Oracle and canonical use identical ddof=1 variance arithmetic on the same float64 data. Max abs error: < 1e-15. Pinned tolerance: `atol=1e-9, rtol=0.0`.

## NaN Convention

Bars 0–9 are NaN (insufficient window or missing first log return). First valid bar: index 10.

## Input

30-bar close price series from GBM (seed=7, σ_step=0.01, S₀=100). Single-row fixture.
