# RV-004 — Model-Free Variance Replication (CBOE Formula)

**Status:** active · v1 (2026-05-08)
**Category:** realized-volatility
**Canonical:** `PythonDataService/app/volatility/vix_replication.py::replicate_expiry_variance`

## What this pins

The CBOE VIX whitepaper per-expiry variance integral `σ²(T)` on 3 BSM-priced synthetic option chains (5–7 strikes each, zero spread).

## Oracle

Direct CBOE formula in Python without calling `replicate_expiry_variance`. Same algorithm, independently coded.

## References

- CBOE VIX 2019 whitepaper "VIX Index Calculation: Step-by-Step"
- Demeterfi, Derman, Kamal, Zou (1999). "More Than You Ever Wanted to Know About Volatility Swaps."

## Formula

```text
F   = K* + e^(rT) · (C(K*) − P(K*))     (put-call parity forward)
K0  = max{K : K ≤ F}
Q   = put_mid  for K < K0
    = call_mid for K > K0
    = (call_mid + put_mid)/2  at K0

ΔK  = (K_{i+1} − K_{i-1})/2  interior; forward/back diff at edges

σ²(T) = (2/T)·Σ(ΔK/K²)·e^(rT)·Q(K) − (1/T)·(F/K₀ − 1)²
```

## Fixture Design

Both `call_mid` and `put_mid` are stored for all strikes. The test passes full `OptionQuote(K, call_mid, call_mid, put_mid, put_mid)` objects so `_select_atm_strike` can use put-call parity to recover the correct forward.

## Agreement

Oracle uses `math.exp`; canonical uses `math.exp` in the same way. BSM-priced chains with zero spread — deviates from the Black-Scholes model only through strike discretization. Pinned tolerance: `atol=1e-6, rtol=0.0`.
