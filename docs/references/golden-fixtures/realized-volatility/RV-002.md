# RV-002 — HF Two-Component Realized Volatility (ABDL)

**Status:** active · v1 (2026-05-08)
**Category:** realized-volatility
**Canonical:** `PythonDataService/app/engine/edge/features_realtime/hf_realized_vol.py::hf_realized_vol_trd252`

## What this pins

ABDL two-component HF realized volatility on 5 synthetic trading days × 4 ETH bars per day (window=3 trading days).

## Oracle

Pure-Python loop grouping bars by trading day, computing intraday squared-return sums + overnight², rolling over 3 days, annualizing × 252/W, then ffilling onto the bar grid.

## Reference

Andersen, Bollerslev, Diebold, Labys (2003). "Modeling and Forecasting Realized Volatility." *Econometrica* 71(2), 579-625.

## Formula

```text
For each day d:
  RV²_d = Σ ln(close_i / close_{i-1})²   (intraday)
          + ln(first_close_d / last_close_{d-1})²  (overnight)

Rolling (W=3 days):
  rv_hf_d = √(sum RV²_{d-W+1..d} × 252 / W)
```

## Agreement

Oracle uses `math.log`; canonical uses `np.log`. Max abs error from float64 accumulation: < 1e-13. Pinned tolerance: `atol=1e-8, rtol=0.0`.

## NaN Convention

First (W-1)=2 trading days (8 bars) are NaN. Each day's value is ffilled onto its 4 bars.

## Input

Timestamps: UTC int64 ms. Sessions: ETH 04:00–20:00 ET. Days 2024-01-02 through 2024-01-08.
