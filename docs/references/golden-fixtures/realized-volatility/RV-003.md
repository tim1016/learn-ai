# RV-003 — IV-RV Basis Conversion (ACT/365 → TRD/252)

**Status:** active · v1 (2026-05-08)
**Category:** realized-volatility
**Canonical:** `PythonDataService/app/volatility/basis.py::convert_iv_act365_to_trading252`

## What this pins

Volatility basis conversion from ACT/365 to TRD/252 convention across 3 pinned cases with varying σ, tenor, and asof date. The NYSE trading-day count N is pinned in the fixture for traceability.

## Oracle

Direct formula: `σ_TRD252 = σ_ACT365 × √((D × 252) / (365 × N))` where N is computed using the same ET-date logic as the canonical.

## ET-Date Alignment

The asof is stored as midnight UTC int64 ms. Both the generator and the canonical convert this to America/New_York before flooring to a date — midnight UTC maps to the previous ET calendar day. This alignment ensures the pinned N matches the canonical's `nyse_trading_days_in_window` call exactly.

## Pinned Cases

| 0.30 | 21 | 2024-01-02 | 13 | 0.3168... |
| 0.25 | 30 | 2024-04-01 | 21 | 0.2483... |

N counts from the ET-date derived from the UTC midnight asof (Jan 1, not Jan 2 for the first two cases).

## Agreement

Oracle and canonical apply identical single-multiplication arithmetic. Float64 rounding agrees to < 1e-16. Pinned tolerance: `atol=1e-9, rtol=0.0`.
