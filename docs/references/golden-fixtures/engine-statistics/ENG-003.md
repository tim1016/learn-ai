# ENG-003 — Trade Statistics

**Status:** active · v1 (2026-05-09)
**Category:** engine-statistics
**Canonical:** `PythonDataService/app/engine/results/statistics.py::compute_trade_statistics`

## What this pins

All standard trade performance metrics — win rate, profit factor, payoff ratio, expectancy, largest win/loss — from `compute_trade_statistics`, validated against an independent formula implementation across 3 synthetic 4-trade cases.

## Oracle

hand_computed — explicit Python formula replicating the same arithmetic from first principles without calling `compute_trade_statistics`.

## Metrics Pinned

| Metric | Formula |
|--------|---------|
| `total_trades` | count of all trades |
| `winning_trades` | count where pnl_pct > 0 |
| `losing_trades` | count where pnl_pct < 0 |
| `win_rate` | winning / total |
| `avg_win_pct` | mean of positive pnl_pcts |
| `avg_loss_pct` | mean of negative pnl_pcts (signed) |
| `avg_trade_pct` | mean of all pnl_pcts |
| `largest_win_pct` | max(pnl_pcts) |
| `largest_loss_pct` | min(pnl_pcts) |
| `profit_factor` | gross_win / gross_loss |
| `expectancy_pct` | avg_trade_pct |
| `payoff_ratio` | avg_win / \|avg_loss\| |

## Reference

Pardo, *The Evaluation and Optimization of Trading Strategies* (2e), §4 (standard performance ratios).

## Input Cases

| Case | Trades (pnl_pct) | wins | losses |
|------|-----------------|------|--------|
| 1 | [+5%, -2%, +3%, -1%] | 2 | 2 |
| 2 | [+10%, +5%, -8%, -2%] | 2 | 2 |
| 3 | [-5%, -3%, +4%, -1%] | 1 | 3 |

## Agreement

Integer metrics: exact. Float metrics pinned at `atol=1e-9, rtol=0.0`. Rationale: the hand-computed oracle evaluates the same deterministic arithmetic (sums and quotients over small integer-valued trade P&L inputs), so rounding differences occur at the float64 ULP level; 1e-9 is several orders of magnitude above the observed max abs error.
