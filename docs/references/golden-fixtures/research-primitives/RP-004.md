# RP-004 — Signal Z-score (Train-period Standardization)

**Status:** active · v1 (2026-05-08)
**Category:** research-primitives
**Canonical:** `PythonDataService/app/research/signal/standardize.py::compute_train_zscore`

## What this pins

Z-score and sign-flipped z-score from `compute_train_zscore`, validated against direct formula application on 50 synthetic bars (35 train, 15 test).

## Oracle

hand_computed — direct formula without calling `compute_train_zscore`:

```text
mu_train    = mean(feature[train_mask])
sigma_train = std(feature[train_mask], ddof=1)  # sample std
z           = (feature - mu_train) / sigma_train
z_flipped   = -z
```

### ddof convention

Both oracle and canonical use **ddof=1** (sample standard deviation). This matches pandas `Series.std()` default. Using numpy's `ndarray.std()` default (ddof=0) produces a ~1.5% divergence for n=35, which was the root cause of a v1 bug corrected before initial commit.

## Reference

Standard z-score normalization. Train/test split convention from standard cross-validation literature.

## Tolerance

`atol=1e-9, rtol=0.0` — exact float64 arithmetic; no transcendental functions.
