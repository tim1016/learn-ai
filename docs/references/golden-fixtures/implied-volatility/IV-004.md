# IV-004 — IV Rank Rolling 60-Day Window

**Status:** active · v1 (2026-05-08)
**Category:** options-pricing
**Canonical:** `PythonDataService/app/research/features/options_features.py::OptionsFeatures.compute_iv_rank`

## What this pins

The rolling IV rank computation `(iv - min) / (max - min)` over a 60-day window with min_periods=30 on an 80-bar synthetic IV series.

## Oracle

Pure-Python loop computing rolling min/max and rank formula without calling `compute_iv_rank` or pandas. Pre-warmup bars (count < min_periods) return 0.5 — matching the canonical's `np.where(denom > 1e-10, rank, 0.5)` behavior when denom=NaN.

## NaN vs 0.5 Behavior

**Important:** bars before `min_periods=30` return **0.5**, not NaN. The canonical uses `np.where(denom > 1e-10, ...)` where `NaN > 1e-10` evaluates to False, yielding 0.5. This is a design choice in the canonical — not a bug.

## Agreement

Oracle and canonical apply identical subtraction+division arithmetic on float64. Pinned tolerance: `atol=1e-9, rtol=0.0`.

## Input

80-bar IV series from a mean-reverting random walk (seed=42, mean=0.20, σ_step=0.012). Single-row fixture (all bars in one wide row).
