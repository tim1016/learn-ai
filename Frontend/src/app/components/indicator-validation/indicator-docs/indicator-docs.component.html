Overview

The indicator table generates OHLCV data plus 6 indicator families in a single request. Data is fetched from Polygon.io and all indicators are calculated server-side using the pandas-ta library — the same formulas used by TradingView's Pine Script built-in indicators.

Data Pipeline

Polygon.io API Fetches OHLCV bars for the requested ticker, date range, and timespan
pandas-ta Calculation Computes all indicators on the full dataset — EMA, Bollinger Bands, Supertrend, RSI, MACD, ADX
JSON Serialization NaN values become null, all floats rounded to 6 decimal places, returned as row dictionaries

Warm-up Buffer

Recursive indicators (EMA, RSI, ADX, MACD) need historical data before the requested window to produce stable values. The system automatically fetches 5 × max lookback extra bars before the requested start date, calculates all indicators on the full dataset, then trims the output to the requested window.

Example For a request with EMA-200, the max lookback is 200. The warm-up fetches 200 × 5 = 1000 extra bars (~3 trading days of minute data) before the start date. This ensures even the longest EMA has fully converged before the output window begins.

Implementation Notes

Calculation Engine

Python pandas-ta library — wraps TA-Lib-compatible implementations in pure pandas

Backend Route

POST /api/indicators/generate-table on the Python FastAPI service

GraphQL Query

generateIndicatorTable — proxied through the .NET backend with Polly resilience

Precision

All indicator values rounded to 6 decimal places; NaN mapped to null

Warm-up Buffer

Fetches 5× max lookback extra bars before the requested window for indicator convergence

ADX tvmode

ta.adx(tvmode=True) — uses TradingView-compatible ADX calculation variant

Data Lab CSV

POST /api/dataset/generate-csv — chunked fetch + 146 dynamic indicators + streaming CSV
